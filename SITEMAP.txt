📊 DATA FLOW DIAGRAM
═════════════════════════════════════════════════════════════════

PLAYER REGISTRATION FLOW:
    
    Player Opens Website
            ↓
    Sees Tournaments (Fetched from DB via API)
            ↓
    Clicks "Register Now"
            ↓
    Fills Registration Form
            ↓
    Submits Form (POST /api/register)
            ↓
    Data Validated (Client & Server)
            ↓
    Stored in Database (registrations table)
            ↓
    Success Message Displayed
            ↓
    Admin Sees Registration in Dashboard

ADMIN TOURNAMENT MANAGEMENT FLOW:
    
    Admin Logs In (POST /api/admin/login)
            ↓
    Session Token Generated & Stored
            ↓
    Accesses Dashboard (GET /api/admin/tournaments)
            ↓
    Tournaments Loaded (With Session Auth)
            ↓
    Admin Creates/Edits/Deletes Tournament
            ↓
    Changes Reflected Instantly
            ↓
    Player Portal Updated Automatically
            ↓
    Players See New Tournament Immediately

🔐 AUTHENTICATION FLOW
═════════════════════════════════════════════════════════════════

LOGIN PROCESS:
    Enter Credentials
            ↓
    Validate (Username exists & Password matches hash)
            ↓
    Generate Session ID
            ↓
    Store in Server Memory (sessions object)
            ↓
    Return Session ID to Client
            ↓
    Client Stores in localStorage
            ↓
    Redirect to Dashboard

AUTHENTICATED REQUESTS:
    Make API Request
            ↓
    Include x-session-id in Headers
            ↓
    Server Validates Session
            ↓
    Session Valid? → Allow Request
            ↓
    Session Invalid/Expired? → 401 Unauthorized
            ↓
    Redirect to Login

LOGOUT PROCESS:
    Click Logout
            ↓
    Clear localStorage
            ↓
    Clear Session (server)
            ↓
    Redirect to Login Page
